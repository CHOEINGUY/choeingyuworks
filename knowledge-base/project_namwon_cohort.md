# 남원 코호트 검진 현장 배정 시스템 - 기술 상세

> 배경 및 도입 동기는 커버레터 참고.

## 배정 알고리즘

- 기본 원칙: 도착 순서 선착순
- 선행 조건 두 가지: 심전도 → 채혈 완료 필수, 인바디 → 채혈 완료 필수
- 조건을 단순하게 유지한 것이 의도적인 선택. 조건이 복잡해질수록 배정이 막히는 케이스가 늘어나기 때문.
- SNSB-C (1시간 이상 소요 검사 별도 로직): 해당 검사실이 비면, SNSB-C 아직 안 한 사람 중 도착 순서 빠른 사람 자동 배정.

## 기술 스택 구성

- AppSheet: 접수/출석 관리, 검사실별 뷰 (계정별 권한 분리), 서명 수집(이미지 → Google Drive 자동 저장), 완료 처리. 전자동의서 URL을 액션으로 구현해 서명 수집.
- Google Sheets: 중앙 DB 겸 연구원들의 예약 관리표 겸용. DB를 직접 건드리는 방식이지만 팀 전원이 이미 익숙하고, 개발 우선순위가 현황판이었기 때문에 선택.
- Google Apps Script: 배정 로직 (선행 조건 체크 + 도착 순서 계산), Google Cloud TTS로 다음 대상자 이름 음성 호명, 현황판 웹앱 구동.
- Firebase (2차 개선): 초기엔 현황판 웹앱이 1초마다 Sheets 데이터를 polling하는 방식이었다. Apps Script에서 데이터 변경 트리거 발생 시 Firebase에 push → 현황판이 실시간 수신하는 구조로 전환. polling → event-driven.

## 현황판 구성

- 화면 1 (대형 TV): 상단 - SNSB-C 검사실 3개(현재 검사자, 경과시간), 중앙 - 다음 대상자 정보, 하단 - 심전도/채혈채뇨/신체기능 검사실별 현재 검사자·경과시간·다음 대상자.
- 화면 2 (보조 모니터): 전체 진행 상황 한눈에.
- 검사 완료 시: 해당 검사실 깜빡이는 애니메이션 + 알림음 + TTS 이름 호명.

## 도입 과정 (연도별)

- 2023년: 부분 도입, 단기 인력 4명
- 2024년: 전 검사실 적용 + TTS 추가, 인력 3명으로 감축
- 2025년: 시스템 판단에 완전 의존 → 운영 시간 1시간 단축 (6시간 → 5시간)

## 실패 사례

**권한 분리 미흡**: 처음엔 모든 검사실 뷰를 다 보이게 구성 → 담당자가 다른 검사실 항목에 입력하는 혼선 발생 → 계정별 뷰 분리로 해결.

**완료 버튼 타이밍 불일치**: 담당자마다 버튼 누르는 타이밍이 달랐다(검사 끝난 직후 vs 대상자 모셔다드린 후 vs 깜빡). 이 시간 차이가 현황판 신뢰도를 떨어뜨리고 자동 배정이 오히려 혼란을 야기. 소프트웨어 강제(화면 잠금, 자동 고정 등)로 해결 시도 → 현장 변수 감당 못함. 결국 검사실 한 곳 한 곳 직접 찾아다니며 시스템 목적과 사용 타이밍 재교육(단기 알바생 포함)으로 해결.
