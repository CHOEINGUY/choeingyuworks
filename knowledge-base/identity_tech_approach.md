# 최인규의 기술 접근 방식과 도구

## 정체성: 바이브 코더

특정 스택을 미리 정해두고 일하지 않는다. 문제를 먼저 구조화하고, 그 문제에 맞는 도구를 찾아서 배우는 방식으로 일한다.

구체적인 흐름:
1. 문제의 구조와 흐름을 먼저 파악한다
2. 프로젝트 폴더를 만들고 Claude Code나 Gemini와 함께 00_개요.md 같은 문서로 구조화한다
3. AI와 함께 어떤 도구가 이 문제에 맞는지 논의하고 분석한다
4. 도구가 정해지면 빠르게 실제로 해본다

## AI 코딩 에이전트 활용 방식

Claude Code, Gemini를 단순 코딩 도구가 아니라 문제 구조화 파트너이자 아카이브 비서로 활용한다. 코드를 짜달라는 게 아니라, 문제를 같이 이해하고 방향을 잡고 기록하는 동료 수준으로 쓴다.

## 주력 도구

**Google 플랫폼 (가장 자신 있음)**
- Google Sheets + Apps Script + AppSheet 조합으로 자동화를 주로 구현한다
- 선택 이유: 클라이언트도 쓰기 쉽고, 세 도구가 잘 연동되고, 무료 범위 내에서 꽤 많은 걸 할 수 있어서
- 실제 외주 프로젝트 다수에서 납품한 주력 스택

**웹 개발 (바이브 코딩)**
- Next.js, React, Vue 등 필요에 따라 AI와 함께 구현
- Vercel, Cloudflare로 배포
- 실제 외주 프로젝트에서 여러 번 납품한 경험 있음

**데이터베이스**
- Firebase, Supabase — 실제 외주 프로젝트에서 사용. 쓸 수 있다.
- Pinecone — 현재 포트폴리오 챗봇 구축하면서 사용. 써봤다 수준.

**기타**
- Python — 목업 데이터 생성 등 필요할 때 가끔 사용
- Git / GitHub — 모든 프로젝트 버전 관리에 사용
- Solapi — 문자 발송 대행 API. 외주 2개 고객사에 납품한 경험 있음

## "어떤 기술 스택 쓰세요?"에 대한 답

특정 스택에 고정돼 있지 않다. 문제를 먼저 파악하고, 그 문제에 맞는 도구를 찾아서 배워서 쓴다. 지금 가장 자주 쓰는 건 Google 플랫폼(AppSheet, Sheets, Apps Script)이고, 웹이 필요하면 AI와 함께 구현한다. 새 도구는 필요할 때 배운다. 그게 지금까지 해온 방식이고, 실제로 동작하는 시스템을 납품해온 방식이다.
