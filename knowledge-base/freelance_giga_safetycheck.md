# 외주 6 - 기가에이전시 안전점검 체크 시스템 (Giga SafetyCheck)

## 한 줄 요약
현장 직원들이 매일 6개 안전항목을 모바일로 제출하고, 근무계획과 매칭해 미제출자를 자동으로 식별·문자 알림까지 발송하는 시스템.

## 기술 스택

- Next.js 16 (App Router), React 19, Tailwind CSS 4, TypeScript (현장 직원용 웹앱)
- Google Apps Script (백엔드, 자동화)
- Google Sheets (안전점검 데이터, 근무계획, 사원 정보)
- AppSheet (관리자 대시보드, 보고서)
- Solapi (SMS 발송 API)
- Vercel 배포

## 시작 배경

크몽으로 연결. 규모는 약 230명의 현장 직원. field-sync(시식 행사)와 비슷한 구조지만 더 큰 프로젝트였다. 핵심 포인트는 본사에서 내려오는 근무계획 엑셀 파일과 안전점검 제출 데이터를 자동으로 매칭하는 것이었다.

## 주요 기능

**현장 직원 (8단계 위저드 UI)**:
- 지점·매장·근무유형 등 기본 정보 입력
- 6개 안전항목 순차 응답

**GAS 자동화 핵심**:
- 제출 즉시 사번+날짜로 근무계획 매칭 → 양방향 물리 참조 기록 (submission_uuid)
- 스마트 스케줄 업로드: 엑셀 업로드 시 기존 데이터를 싹 지우지 않고 메모리에서 변경된 부분만 덮어씀. 기존 제출 기록 보호.
- 매월 1일 새벽 2시, 2개월 이상 지난 데이터 자동 아카이빙 → 메인 시트 무게 유지
- 초고속 전체 매칭: 기존 233초 → 7초로 단축 (메모리 기반 일괄 처리, 이미 매칭된 행 스킵)

**SMS 자동 발송**:
- 단건: AppSheet 봇 웹훅에서 수동 트리거
- 일괄: 시간 기반 트리거 하루 2회 — 당일 미제출자 전원 수집 후 Solapi API로 1회 일괄 발송
- 개발/검증 시 TEST_MODE로 Discord 웹훅에 발송해서 실제 문자 낭비 없이 테스트

**권한 기반 보고서**:
- 관리자: 전체 현황 조회
- 지점 관리자: 담당 지점만 출력
- 점검 현황 리포트를 Google Drive에 새 스프레드시트로 자동 생성 후 URL 반환
